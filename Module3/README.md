## 🛠️ Compilation and Execution Guide

### 1. Compile All Main Source Files
Open your terminal in `d:\New folder\Module3` and run:
```powershell
javac *.java
```
This compiles all 40 root `.java` files simultaneously and creates their corresponding `.class` files.

### 2. Compile and Run Java Modules (Exercise 34)
To prevent the IDE's Java Language Server from incorrectly treating the entire root workspace as part of the `com.utils` module (which causes "Must declare a named package" and import errors on the other standalone files), the modular exercise files have been saved with `.txt` extensions.

To compile and run Exercise 34, rename them to `.java` and compile:
```powershell
cd Exercise34_Modules

# Rename module and source files to .java
mv com.utils\module-info.java.txt com.utils\module-info.java
mv com.utils\com\utils\StringUtils.java.txt com.utils\com\utils\StringUtils.java
mv com.greetings\module-info.java.txt com.greetings\module-info.java
mv com.greetings\com\greetings\Main.java.txt com.greetings\com\greetings\Main.java

mkdir mods\com.utils
mkdir mods\com.greetings

# Compile com.utils module
javac -d mods\com.utils com.utils\module-info.java com.utils\com\utils\StringUtils.java

# Compile com.greetings module using the module-path
javac --module-path mods -d mods\com.greetings com.greetings\module-info.java com.greetings\com\greetings\Main.java

# Run the modular application
java --module-path mods --module com.greetings/com.greetings.Main

# Optional: Restore back to .txt to clear IDE conflicts
mv com.utils\module-info.java com.utils\module-info.java.txt
mv com.utils\com\utils\StringUtils.java com.utils\com\utils\StringUtils.java.txt
mv com.greetings\module-info.java com.greetings\module-info.java.txt
mv com.greetings\com\greetings\Main.java com.greetings\com\greetings\Main.java.txt
```

## 💡 Important Design Implementations & Fallbacks

- **JDBC (Exercises 31-33)**: Targets SQLite databases (`students.db` and `bank.db`). Uses standard `java.sql` classes, compiling natively without third-party JARs.
- **Pattern Matching for Switch (Exercise 30)**: Designed to compile out-of-the-box in Java 17 using standard pattern-matching `instanceof` logic. A commented-out Java 21 `switch` implementation is included for native execution on newer JDKs.
- **Virtual Threads (Exercise 40)**: Implements dynamic reflection loading to run on JDK 21+ Virtual Threads executor if available, falling back gracefully to a Cached Thread Pool on JDK 17 to maintain compilation safety.
